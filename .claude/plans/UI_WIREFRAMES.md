## 공통 App Shell

7개 화면 모두 동일한 프레임을 사용한다. **상단 글로벌 바**(항상 고정)와 **좌측 네비**(6개 항목 고정)는 모든 화면에서 글자 그대로 동일하며, 좌측 네비에서 **현재 화면만 `●`** 로 강조한다. 본문(MAIN CONTENT)만 화면별로 채운다.

- **상단 글로벌 바**: `☰ 햄버거` · `HANBIT 로고` · 시장별 모드 뱃지(`KR 🔴LIVE` · `OS 🔴LIVE` · `FUT 🟡PAPER·HKEX`) · `WS● 연결` 상태 · `서버시계(09:31:05 KST)+시장세션(美장중 HK장중)` · `🔴 KILL` 글로벌 킬스위치.
- **좌측 네비(6개 고정)**: Overview / Positions / Orders / Charts / Strategies / Risk. 현재 화면만 앞에 `●`.
- 폭은 약 100칸. 외곽 박스는 반드시 닫히게(`┌─┬┐ │ ├┼┤ └┴┘`). CJK 2칸 폭 때문에 완벽한 우변 정렬은 포기하고 레이아웃 전달을 우선.

**핵심 구성요소·인터랙션**
- 상단 KPI 행 4칸: 활성전략수(4) · 열린주문수(7) · 포지션수(12) · 위험상태 요약(MDD -2.1% / 한도 -8%, Risk 화면 링크)
- 통화별 카드 3개(₩ 국내주식 🔴LIVE / $ 해외주식 🔴LIVE / HK$ 해외선물 🟡PAPER): 각 총평가·매입·손익(금액+율, 등락색)·예수금/가용증거금. 통화기호·모드 뱃지를 카드마다 명시하여 절대 합산하지 않음을 시각화
- 실거래/모의 분리 손익 영역: 🔴LIVE와 🟡PAPER를 물리적으로 분리한 두 블록(실현/미실현/일중율). '격리·미합산' 라벨로 버킷 격리 강조
- 시장별 분배 도넛(KR/OS/FUT 비중 %): 환율(₩1,318/$, ₩171/HK$)·환산시각(09:31:00 KST) 표기. /portfolio/allocation 응답 구동
- 일중 PnL 스파크라인: 세션·통화별 3행(₩KR / $OS / HK$F)으로 분리, 각 행에 시장세션(美장중/HK장중)·등락율 표시
- 기준통화(₩) 환산 총액 카드: ★별도 카드로만★ 노출(≈₩110,940,000). FX·환산시각·'참고용 합산' 명시로 통화별 분리 원칙과 충돌 방지
- WS 수신 상태/seq 표시줄(pnl·balance·account_pnl, seq 8842)로 실시간 갱신·누락 감지 가시화
- 상단바 글로벌 킬스위치(🔴 KILL)·시장별 모드 뱃지·WS 연결·서버시계+세션은 App Shell 고정 요소

**데이터 피드 (§6 계약)**
- REST GET /account/summary — KPI(활성전략·열린주문·포지션수)·통화별 총평가/매입/손익(금액·율)·실거래/모의 분리 손익
- REST GET /account/balance — 통화별 예수금/가용증거금(₩·$·HK$, mode·market 분리)
- REST GET /portfolio/allocation — 시장별 분배 도넛(KR/OS/FUT 비중)·환율·환산시각·기준통화(₩) 환산 총액 카드
- WS topic: pnl — 일중 PnL 스파크라인(세션·통화별) 실시간 갱신
- WS topic: balance — 통화별 예수금/가용증거금 실시간 갱신
- WS topic: account_pnl — 통화별 카드 손익·실거래/모의 분리 손익·위험상태 KPI(account_tracker on_account_pnl_change 구동, mode/market/seq 포함)

**핵심 구성요소·인터랙션**
- 헤더는 모드색으로 위험을 즉시 신호: LIVE=🔴빨강 "실거래 주문 확인", PAPER=🟡노랑 "모의 주문 확인". 우측 `expires_in 0:27` 카운트다운 — 0:00 도달 시 견적 만료·버튼 비활성 후 [재견적] 전환.
- 견적 블록: market·symbol·side·qty·price + est{notional, fee, post_trade_exposure, currency}. side는 BUY🟢 / SELL🔴 색 구분.
- 리스크 사전체크 요약 한 줄(✓모드 ✓소액상한 ✓누적명목 ✓일손실 ✓킬스위치). 하나라도 실패 시 ⛔+사유, [확정] 영구 비활성. 예시는 명목 $48 < 소액상한 $50/건 → ✓.
- LIVE 2단계 게이트: 체크박스 2개(금액·수량·side 재확인 / 실거래(LIVE) 확정) **모두 ☑** 여야 [확정(commit)🔴] 활성. 하나라도 ☐면 회색 비활성.
- 버튼: [취소](항상 활성, 모달 닫고 토큰 폐기) / [확정(commit)🔴](2체크+미만료 시에만 활성).
- PAPER 변형: 헤더 노랑, 재확인 체크박스 없이 단일단계 — 열리면 [확정] 즉시 활성.

**데이터 피드 (§6 계약)**
- 모달 진입: `POST /orders/quote` {market,symbol,side,qty,price} → {quote_id, est{notional,fee,post_trade_exposure,currency}, risk_checks[], expires_in, confirm_token}. 카운트다운은 expires_in 구동.
- 확정: `POST /orders/commit` {confirm_token} (LIVE는 2체크 완료 후에만 호출). 만료/위변조 토큰은 4xx → [재견적] 유도.
- 체결 스트림: commit 성공 후 `WS fill` → {ordno, exec_qty, exec_price, status, unfilled_qty} push로 Positions/Orders 실시간 갱신.
