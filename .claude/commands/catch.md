# /catch — 세션 컨텍스트 복구 + 다음 작업 연속 (한빛)

`/clear` 직후 새 세션이 **대화 기억 없이** 이어가기 위한 부트스트랩. 대화 기록 대신
**커밋된 텍스트**(`.claude/plans/STATUS.md` → 가리키는 설계문서 → git 커밋 메시지)를 진실원으로 신뢰한다.

> 이 명령은 두 경로로 들어온다: ① `/recycle` 이 watcher 를 통해 자동 전송(`/catch <다음 작업>`),
> ② 사용자가 새 세션에서 직접 호출. 둘 다 동일하게 동작한다.

이어서 할 작업: **$ARGUMENTS**  (비어 있으면 STATUS '다음' 1순위를 자동 선택)

---

## Step 1. 핸드오프 단일 진실원 로드 (필수)

```bash
git branch --show-current
git log --oneline -6
git status --short
```

- **`.claude/plans/STATUS.md` 전체를 읽는다** — 세션 핸드오프의 단일 진실원(SSOT). 특히 맨 위 안내, **완료된 것**,
  **재개 방법**, 그리고 맨 아래 **## 다음**(1순위 = 정확한 다음 행동).
- STATUS 가 가리키는 설계문서(`.claude/plans/2026-06-20-통합계획서.md M2`, `.claude/plans/2026-06-20-통합계획서.md §8` 등)는 **필요한 섹션만** 읽는다(전체 X).
- 메모리 스토어는 이 환경에서 깨져 있다 — 대화/메모리에 기대지 말고 STATUS + 커밋만 신뢰한다.

## Step 2. 진행 상황 종합

커밋 메시지(시간순) + STATUS 를 종합해 다음을 1~3줄로 정리:
1. **목적/현재 위치**: 어느 마일스톤(M0~M?)·브랜치·마지막 커밋까지 와 있는지.
2. **다음 행동**: STATUS '다음' 1순위 = 정확한 다음 한 가지(파일·실행 명령·재개 스크립트).
3. **블로커**: 1순위가 **사용자 액션 대기**(예: 외부 권한 신청 등)면 명시. 막혀 있으면 2순위로 넘어갈지 판단.

## Step 3. 다음 작업 선택 + 시작

- **`$ARGUMENTS` 가 있으면** → 그것이 이번 작업. 관련 파일/섹션만 읽고 즉시 시작.
- **비어 있으면** → STATUS '다음' 1순위를 선택. 단 **1순위가 사용자 액션으로 블로킹**돼 있고 코드로 못 풀면,
  그 사실을 한 줄 보고하고 **블로킹 안 된 다음 후보(2순위)** 로 진행하거나, 후보가 없으면 사용자에게 무엇이
  필요한지 묻고 멈춘다(임의로 막힌 작업을 우회하지 않는다).

## Step 4. 출력 형식

```
## 컨텍스트 복구 (STATUS.md 기준)
- 위치: <마일스톤/브랜치/마지막 커밋>
- 완료: <핵심 1~2줄>
- 다음(1순위): <STATUS '다음' 1순위> [블로커: <있으면>]

→ 이번 세션 작업: <지금 시작할 것 한 줄>
```

그다음 **즉시 작업을 시작**한다(추가 확인 질문 없이 — 단 Step 3 의 블로킹 예외는 제외).

---

## 핵심 원칙

- **최소 컨텍스트 + 즉시 작업**: STATUS + 커밋으로 맥락 복구, 설계문서는 필요한 섹션만.
- **STATUS 가 인덱스**: 상세 설계/로그는 `.claude/plans/` 문서에 있고 STATUS 는 "다음 1순위 + 가리키는 곳"이다.
- **막힌 작업 우회 금지**: 사용자 액션 블로커는 보고하고 다음 후보로 가거나 멈춘다.

$ARGUMENTS
